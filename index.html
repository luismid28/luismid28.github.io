DIFERENCIA ENTRE DATO E INFORMACIÓN

Un dato es la unidad mínima de información que representa, un valor, hecho o característica, un dato por sí solo no tiene significado pero cuando se le agrega un contexto el dato se convierte en información, por ejemplo: el número 24 no significa nada, pero si le agregas Juan tiene 24 años, ahora es información.

CONCEPTO DE ESTRUCTURA DE DATOS

Las estructuras de datos son una forma de organizar y almacenar datos para poder usarlos, acceder a ellos y modificarlos de manera eficiente. Un ejemplo son los arreglos.

TIPOS DE DATOS Y ALGORITMOS

Muchas cosas pueden ser datos, un número, una letra, un símbolo, incluso una pequeña señal eléctrica, a esa señal se le llama bit y es la base de la informática. Los bits solo pueden representar los valores 0 y 1, un byte es un conjunto de 8 bits, con los bytes puedes representar caracteres. Las computadoras cuentan en base a 2.

Un algoritmo es un conjunto ordenado de pasos que describen cómo resolver un problema o realizar una tarea. Los algoritmos permiten convertir datos en información mediante distintos procesos por eso son tan importantes.

En la tercera clase vimos la introducción a la notación Big O.

NOTACIÓN BIG O

La notación Big O se usa para describir qué tan eficiente es un algoritmo, indicando cómo crece el tiempo o la memoria que necesita cuando aumenta la cantidad de datos de entrada.

Tiempo constante O(1):Es un tipo de comportamiento donde el algoritmo realiza una cantidad fija de pasos , sin importar cuántos datos existan.

Crecimiento lineal O(n):Es cuando el número de pasos del algoritmo crece al mismo ritmo que los datos.

Crecimiento cuadrático O(n²):Es cuando el algoritmo realiza operaciones por cada dato con todos los demás datos.

Crecimiento logarítmico O(log n):Es cuando el algoritmo reduce el problema en cada paso, normalmente dividiéndolo en partes más pequeñas.

HERRAMIENTAS Y ENTORNOS DE DESARROLLO

Un entorno de desarrollo es importante para la calidad del software porque ayuda a crear algoritmos claros y correctos, permite trabajar con datos de forma segura y ordenada, y facilita encontrar errores antes de que el programa se use en la vida real. Gracias a estas herramientas, el desarrollador puede probar, corregir y mejorar el software sin romper lo que ya funciona, logrando sistemas más estables, confiables y fáciles de mantener.

Un entorno de desarrollo es un conjunto de herramientas que se usan para crear software, y permite escribir código, diseñar algoritmos, manejar datos, probar programas y corregir errores. Gracias a él, los desarrolladores pueden trabajar de forma más ordenada y eficiente, asegurando que el software funcione correctamente antes de ser utilizado. También facilita el trabajo en equipo.

A través de los lenguajes de programación se crean algoritmos que procesan datos y resuelven problemas, haciendo posible el desarrollo de programas, aplicaciones y sistemas que usamos todos los días. Algunos ejemplos son Python, que se suele utilizar para análisis de datos, inteligencia artificial y automatización por su simplicidad, Java que se usa mucho para aplicaciones empresariales y aplicaciones Android por su estabilidad y portabilidad y JavaScript, que se emplea principalmente para el desarrollo de páginas web interactivas y aplicaciones web, ya que permite trabajar tanto del lado del navegador como del servidor.

Estos lenguajes se relacionan con las estructuras de datos porque todos permiten organizar información de forma eficiente. En Python se usan listas, diccionarios y conjuntos para trabajar fácilmente con datos, en Java son comunes los arreglos, listas, pilas y colas para construir aplicaciones más estructuradas y en JavaScript se emplean arreglos y objetos para almacenar y manipular datos en aplicaciones web, lo que ayuda a que los algoritmos funcionen de manera correcta y ordenada.

Algunas de las herramientas más importantes a la hora de crear proyectos son las siguientes:

-Editores de código y los entornos de desarrollo integrados (IDE): permiten escribir, organizar y ejecutar el código, además de detectar errores mientras se programa.

-Git y GitHub: Se utilizan para controlar versiones del software, guardar cambios y colaborar con otras personas sin perder el trabajo previo.

-Las herramientas de depuración: Sirven para encontrar y corregir errores en los algoritmos, analizando cómo se comportan los datos durante la ejecución.

-Gestores de dependencias: facilitan la instalación y actualización de librerías necesarias para que el programa funcione correctamente, manteniendo el proyecto estable y ordenado.

Elegir el entorno de desarrollo correcto es importante para la eficiencia porque permite trabajar más rápido, cometer menos errores y manejar mejor los algoritmos y los datos. Un buen entorno ofrece herramientas adecuadas para escribir, probar y depurar el software sin perder tiempo en tareas innecesarias. En cambio, elegir herramientas incorrectas puede causar problemas como errores difíciles de detectar, pérdida de tiempo, mala organización del código y dificultades para trabajar en equipo, lo que reduce la calidad del software y hace el desarrollo más lento y costoso.

Las buenas prácticas en el uso de herramientas de código ayudan a mantener el software ordenado y fácil de mantener. Algunas de las más importantes son usar correctamente el control de versiones como Git para guardar cambios frecuentes, escribir código claro y bien organizado en el editor o IDE, aprovechar las herramientas de depuración para detectar errores a tiempo y mantener actualizadas las dependencias del proyecto. También es buena práctica configurar el entorno de desarrollo de forma adecuada y usar siempre las mismas herramientas dentro de un equipo, ya que esto mejora la colaboración y reduce errores. Otra cosa que ayuda es la documentación para que se sepa que se esta haciendo y se lleve un registro del funcionamiento y realizar prueebas constantemente.
